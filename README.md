# Design-pattern

## 介绍
以图解设计模式 ([日]结城浩) 为主线做的笔记

**注意**

>  作者在书中强调了"**不要使用具体类来编程，要优先使用抽象类和接口来编程**"。

我虽然没有达到作者的那种境界，但是对这句话存在十分敬畏。

**GoF对设计模式的分类**

1. 创建型设计模式

    2. Builder
    3. Factory Method
    4. Prototype
    5. Singleton
2. 结构型设计模式
    1. Adapter
    2. Bridge
    3. Composite
    4. Decorator
    5. Facade
    6. Flyweight

3. 行为型设计模式

    2. Interpreter
    3. command
    4. Iterator
    5. Mediator
    6. Memento
    7. Observer
    8. State
